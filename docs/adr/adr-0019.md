# ADR-0019: Checkpoint/恢复机制设计

**状态:** Accepted
**日期:** 2026-05-11
**Phase:** Phase 10 — Production Hardening

## 决策

- 每个 stage 完成后写入 `data/{target_id}/memory/checkpoint_{stage}.json`
- Checkpoint 内容：`stage`、`cursor`（偏移量/key）、`processed_ids`（已处理 event ID 集合）、`saved_at`
- 重启时从对应 stage checkpoint 恢复，已处理的 event 跳过
- 错误分为 3 类：transient（重试 3 次，指数退避）、data（跳过并记录）、fatal（停止运行）
- transient 3 次失败后升级为 fatal

## 理由

- 文件驱动，无额外基础设施依赖（v1 不引入消息队列）
- `processed_ids` 用 set 而非计数，支持幂等
- 错误分类使恢复策略与错误类型匹配

## 影响

- `src/news_sentry/core/checkpoint.py` 新增 `StageCheckpoint` + `CheckpointManager` + `ErrorType`
- bounded run 需集成 checkpoint save/load
