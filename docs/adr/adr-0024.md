# ADR-0024: Schema 版本治理策略

> 日期: 2026-05-16
> 状态: Accepted
> 影响: schemas/ 全部 JSON Schema 文件

## 决策

采用"稳定即终态"策略：v1.x 阶段所有 schema 保持向后兼容，不做 breaking change。

## 背景

`schemas/` 目录包含 18 份 JSON Schema (2020-12)，每份均已有 `$schema` 和 `$id` 字段。项目需要一个版本治理策略，明确何时可以修改 schema、如何处理不兼容变更。

## 决策内容

1. **版本标识**: 每份 schema 通过 `$id` URI 唯一标识（如 `https://news-sentry.local/schemas/newsevent.schema.json`），不引入额外的独立版本号字段
2. **兼容性承诺**: v1.x 阶段所有变更必须向后兼容——只能添加可选字段，不能删除或重命名已有字段
3. **Breaking change 流程**（预留给 v2.0.0）:
   - 创建新 schema 文件（如 `newsevent.v2.schema.json`）
   - 旧 schema 保留至少一个大版本周期
   - ConfigLoader 通过 `$id` 路由到正确版本
4. **校验绑定**: 配置 YAML 头部 `# Schema:` 注释指向对应 schema 文件名，ConfigLoader 加载时执行校验

## 不做的事

- 不引入自动化 schema 版本迁移工具（YAGNI）
- 不为 schema 添加 `version` 字段（`$id` 已满足标识需求）
- 不引入 schema registry（项目规模不需要）

## 现状确认

18 份 schema 全部已包含：
- `$schema: "https://json-schema.org/draft/2020-12/schema"`
- `$id: "https://news-sentry.local/schemas/<name>.schema.json"`
