# ADR-0025 — 修订 ADR-0010：Web API + 前端作为可选部署模式

| 属性 | 值 |
|---|---|
| **状态** | Accepted |
| **日期** | 2026-05-20 |
| **决策者** | 项目用户 |
| **关联 ADR** | ADR-0010（被修订）、ADR-0012（Python 实现语言）|
| **关联 Phase** | Phase 22 (API Gateway)、Phase 49.5 (App Productization) |

---

## 背景

ADR-0010（2026-05-09）明确禁止 FastAPI 和任何前端框架，规定 News Sentry 的可视化终态为 Obsidian Markdown + 飞书/邮件/Webhook 推送。

自 Phase 22（2026-05-12）起，项目实际上引入了 FastAPI REST API 服务器（`api_server.py`），随后在 Phase 49.5 中扩展为包含完整的 Vanilla JS SPA 前端（`static/` 目录），服务于 Dashboard、事件查询、用户管理、设置等场景。

现状与 ADR-0010 之间存在事实矛盾，需要正式修订架构决策。

---

## 决策

**ADR-0010 中关于"禁止 FastAPI"、"禁止 web 服务"、"禁止前端目录"的限制条款被正式废除。**

### 新定位

1. **FastAPI + Vanilla JS 作为可选部署模式**
   - FastAPI 通过 `[api]` optional extras 控制（`fastapi` + `uvicorn`）
   - 核心 CLI `python -m news_sentry.cli run` 无需 FastAPI 即可运行
   - `static/` 目录为正式支持的前端资源目录

2. **保留 ADR-0010 的核心理念**
   - Obsidian Markdown + 飞书/邮件/推送 仍为默认可视化路径
   - News Sentry 仍然是一个 CLI-first 的 Agent Skill Pack
   - Web UI 是辅助交付载体，不是核心领域模型
   - 不引入 React / Vue / Svelte / Angular 等重型前端框架

3. **前端技术栈约束**
   - 仅使用 Vanilla JS ES modules
   - Chart.js 作为唯一外部图表库（CDN）
   - 不引入 Node.js 构建工具链（webpack/vite/babel）
   - 前端资源通过 FastAPI StaticFiles 挂载

### 重定向规则（更新）

任何"需要前端"的诉求：
1. 能否用 Obsidian Dataview 查询满足？→ 优先 Obsidian
2. 能否用推送消息通知满足？→ 优先推送 Skill
3. 能否用 Output Skill 生成静态 Markdown 报告满足？→ 优先静态输出
4. 以上不满足时 → 可以在 `static/` 中添加 Vanilla JS 页面

---

## 舍弃的选项

| 选项 | 拒绝原因 |
|---|---|
| 直接废除 ADR-0010 | ADR 应通过修订而非删除来演进；废除会丢失原始决策背景 |
| 强制保留 ADR-0010 所有限制 | 与项目现状矛盾，且 Web UI 在 Phase 49.5 中已被验证为用户价值 |
| 引入 React/Vue 等重型框架 | 维护成本超出项目定位，与"简单至上"原则冲突 |

- 消除架构文档与代码现状的矛盾
- 为 FastAPI 服务器和前端代码提供正式的架构合法性
- 保持项目 CLI-first 的核心定位不变
