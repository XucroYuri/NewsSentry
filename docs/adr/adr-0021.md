# ADR-0021: 信源矩阵 13 维分类框架 + 多层浏览器兜底架构

**状态:** Accepted
**日期:** 2026-05-11
**Phase:** Phase 12 — Source Matrix

## 决策

1. 意大利信源按 13 个维度分类（A-M），每个维度至少 2 个可通信源
2. 采集方式分 RSS/API/OpenCLI 三类，优先级：RSS > API > OpenCLI
3. 社媒 KOL 账号分 L1/L2/L3 三级，对应 active/semi-active 两种监控模式
4. 浏览器采集采用 3 层降级架构：OpenCLI Bridge → Playwright MCP → Computer Use
5. 信源生命周期状态机：active → degraded → dead → archive
6. 自进化机制：健康自检 + 热点发现 + KOL 清单扩展

## 理由

- 13 维框架覆盖意大利政治/经济/外交/安全/司法/社会/科技/环境/移民/文化/宗教/涉华/其他全景
- RSS 优先原则确保采集阶段零 Token 消耗，API 其次，OpenCLI 仅做兜底
- 3 层浏览器兜底确保 OpenCLI Bridge 不可用时采集不中断
- 状态机自动管理信源健康，减少人工干预
- 自进化机制使信源矩阵可持续扩展
