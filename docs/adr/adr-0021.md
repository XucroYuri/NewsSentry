# ADR-0021: 意大利信源矩阵 13 维分类框架

**状态:** Accepted
**日期:** 2026-05-11
**Phase:** Phase 12 — Multi-Source Expansion

## 决策

1. **13 维分类框架**：采用 A-M 13 个维度组织全部意大利信源，M 维度作为 Other 兜底
2. **三优先原则**：RSS 优先 -> API 其次 -> OpenCLI 兜底，采集阶段零 Token 消耗
3. **多层兜底**：浏览器采集采用 Layer 1 (OpenCLI Bridge) -> Layer 2 (Playwright MCP) -> Layer 3 (Computer Use) 三层降级
4. **自进化机制**：信源矩阵内建 health audit、热点发现、KOL 列表自扩展三个循环
5. **通知去平台化**：所有告警走 Hermes Agent 信息通道，不硬编码飞书/钉钉等具体平台

## 理由

- 驻外记者站需要全方位监控目标国家舆情，覆盖政治、经济、外交、安全、司法、社会、科技、环境、移民、文化、宗教、涉华等全部维度
- 意大利 reference target 现有 14 个 RSS 信源，全部为传统新闻媒体，缺少社媒、KOL、专业委员会信源
- 部分信源无 RSS feed，需要 OpenCLI Browser Bridge 或 API 方式进行采集
- 采集阶段零 Token 消耗确保成本可控，AI 算力集中在 Filter/Judge 高价值环节

## 替代方案

- 方案 B：仅扩展 RSS 信源，放弃社媒和 KOL -> 被拒绝（覆盖维度不足）
- 方案 C：使用第三方舆情平台 API 替代自建采集 -> 被拒绝（成本不可控、信源不透明）

## 影响

- `config/sources/italy/` 目录扩展三倍，新增 api/、opencli/、social/ 子目录
- `SocialKOLCollector` 从 Phase 6 stub 升级为真实 Bridge 采集器
- Docker 镜像增加 Chromium + Xvfb + Playwright + Node.js 依赖，镜像体积约 +800MB
- 评估集构建推迟到 Phase 13
