# ADR-0020: 多 Agent 编排模式

**状态:** Accepted
**日期:** 2026-05-11
**Phase:** Phase 11 — Intelligence Deepening

## 决策

- v1 提供两种编排模式:SequentialOrchestrator(默认)和 ConcurrentOrchestrator(多信源并行采集)
- Judge 阶段在所有模式下保持顺序执行(依赖 Filter 结果)
- 不引入 Celery/RabbitMQ(v1 保持文件驱动)
- 所有编排模式保留人工介入点:review queue、重大事件确认、publish gate

## 理由

- Sequential 模式保持现有行为,风险最低
- Concurrent 模式解决多信源场景的吞吐瓶颈
- 文件驱动保持 v1 零基础设施依赖原则
- 人工介入点对齐 Iron Man 套装原则

## 影响

- `src/news_sentry/core/orchestrator.py` 新增 `PipelineOrchestrator` + `OrchestratorMode`
- `config/targets/{target_id}.yaml` 的 `pipeline` 节可声明编排模式
